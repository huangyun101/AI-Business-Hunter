# 第七章 技术层面：每个阶段该会什么、不该学什么

程序员做副业有个独特的陷阱：**用学习代替经营**。遇到不确定就去学个新框架，收入没起色就归因于"技术还不够"，因为学习是我们最熟悉、最有安全感的动作。说实话我自己也这么干过。所以这一章反着写：先讲不该学什么，再按阶段讲该会什么。

## 7.1 总原则：技术是成本项，不是资产项

在副业的账本上，你写的代码本身不值钱，**代码接住的需求才值钱**。社群里一位从手搓代码转向模板建站的圈友，心路历程是这条原则最生动的注脚。

他一开始所有网站全部手搓，从按钮到交互逻辑，看着 AI 一行一行写代码，"很神奇，但也很低效"。后来学了刘小排的课才发现，成熟玩家都是基于模板快速上站的。他咬牙买了两个模板，"点击付款的时候心都在滴血，这玩意怎么那么贵"，结果第一次用模板重构网站只花了 11 天，后来一天就能上线，快的话几个小时。他的原话："我之前过的都是什么苦日子。"

模板贵不贵？对程序员的自尊心来说很贵，对生意来说便宜得不像话。**能买的不写，能抄的不设计，能用 AI 生成的不手写**。省下来的时间去做代码替代不了的事：找需求、做流量、聊客户。

配套的第二条原则来自一位重度 AI 工作流玩家：**用最好的模型，别为了省钱用差的**。顶级模型和差模型的差距不是 token 单价，是方向性错误的返工成本。"用差模型省的钱，远不如用好模型省的时间值钱。"工具订阅费是副业里回报率最高的开销之一。

## 7.2 入门阶段（月入几百~几千）：会用，就够了

这个阶段的技术清单短得令人失望：

- **一套 AI 编程工作流**。参考王马扎的组合：ChatGPT 做架构师设计逻辑，Claude Code、Codex 写代码和做 Code Review。你的角色从"写代码的"变成"验收代码的"。
- **你接单方向上的存量技能**。第四章说过，从真实单子倒推：接 Web 单就是 HTML/CSS/响应式/WordPress 这类实用组合；做部署代做就是 Linux 基础加看日志排错的能力。全是你已经会的东西。
- **一个能收钱的动作**。国内：微信/支付宝收款码加一张手写的报价单就够了。别在这个阶段研究支付网关。

不该学的：任何"为了以后可能用到"的新语言、新框架。入门期每一小时都该花在离钱最近的动作上。

## 7.3 中阶阶段（月入几千~几万）：把上线变成流水线

中阶做的是产品和流量资产，技术上的核心命题是：**把"从想法到上线"的时间压缩到小时级**。赫兹（靠 AI 网站出海赚到第一个一万刀）的八步建站流程是现成的标准答案，他一套下来 3~5 小时上线一个网站：

1. 复制模板（他用 MkSaas）→ 推到 GitHub → 本地跑起来；
2. 配好环境变量（数据库、登录）；
3. **先部署上线**（Vercel），这步要尽早做，避免功能做了一堆，部署时才发现服务端跑不通；
4. 改落地页，围绕核心关键词生成文案，注意关键词密度；
5. 接统计：Google Analytics + Plausible + Clarity 三件套；
6. 绑域名、提交搜索引擎（Google/Bing/Naver），域名早上线可以早拿排名；
7. 打磨功能、接入 API，参考竞品在首页加 AI 能力；
8. 接支付，测试环境跑通后再提交审核。

注意这个顺序的反直觉之处：**部署第 3 步就做，功能第 7 步才打磨**。和公司里"开发完再上线"完全相反，因为这里上线的目的是尽早开始积累域名权重和收录。

几个环节的补充细节：

- **部署链路**：哥飞给的标准组合是域名 → Vercel 部署 → Cloudflare 做 CDN（加速加隐藏源站 IP 双重作用）→ 提交 Google Search Console → 接统计。他开源的 Gitbase 模板对爬虫非常友好，新站提交后约 3 小时就能收录，不懂 Next.js 也能在上面改出多语言和支付。
- **统计三件套的分工**：GA 看流量来源，Plausible 轻量看趋势，Clarity 看用户行为录屏。第三章说过，有人靠 Clarity 录屏在上线头几天修掉了好几个 bug。
- **支付（出海）**：Stripe 是最稳的选择，支持一次性付款和订阅两种模式。开发流程：注册后默认 Sandbox 测试模式 → 在 Developer-API Keys 拿 Secret key 和 Publishable key 存进环境变量 → 本地开发时用端口转发地址配置 Webhook Endpoint 调试事件 → 全流程跑通再切 Live。两个坑要提前知道：一是 Stripe 开通支付宝/微信收款审核复杂，且只支持一次性付款、不支持订阅，面向中国用户的订阅产品要单独找客服特批；二是没有海外公司的个人开发者，可以用 Paddle 替代，它对开发者地区和资质的要求更灵活，Vercel 模板中心里 Stripe 和 Paddle 都有现成的 Next.js 订阅模板可以直接 fork。
- **国内支付**：微信小程序/公众号生态走微信支付的官方流程，个人主体受限时，常见做法是挂靠有资质的服务商或走虚拟商品平台代收。细节随政策变化快，动手前查最新规则。

不该学的：自建 Kubernetes、自研支付对账系统、手写 SSR 框架。Vercel 一把梭有它撑不住的那天，但那天你的月收入早就过几万了。

## 7.4 高阶阶段（月入几万以上）：架构为复利服务

高阶玩家的技术决策，判断标准从"能不能跑"变成"能不能复利"。三个来自实战的准则。

**一、一切服务 API 化**。一位管理着一堆自动化工具的圈友的实践：构建项目时优先调用外部 API 处理复杂问题（比如各平台的风控，自己处理费力且脆弱，外部方案随时可替换）；同时自己写的每个服务都对内暴露 API。他的工具之间能互相调用，信息处理服务调用视频下载 API，备忘录服务调用总结 API。**每做一个新产品，旧产品都变成它的积木**。这正是第六章矩阵曲线"时间递减、收益叠加"的技术底层。

**二、业务数据集是你唯一的技术护城河**。一位做内容审核业务的圈友说得透彻：模型迭代太快，今天的模型选型三个月后就过时，但你沉淀的业务测试集（他的场景里是高风险内容数据集）是核心资产。有了它，你能随时像调参数一样评测新模型，快速平替更便宜的方案。他从 GPT-4o 切到 Gemini 2.0，成本降到五分之一，敢切的底气就是那套测试集。**模型是租的，评测集是你的。**

**三、成本工程：给 AI 拆岗位**。一位做数字员工产品的操盘手踩过的坑：一开始幻想一个强模型包打天下，结果一两周 API 费用就爆了，单日成本高到扛不住。后来想明白，数字员工和真人团队一样要拆岗位：抓信息、跑固定接口的活用便宜模型；审核、规划、方案判断用强模型；写代码用擅长代码的模型。**岗位拆开，成本才压得住，产品也更好卖**。这和 7.1 的"用最好的模型"不矛盾：探索期用最强的保方向，量产期按岗位配置保毛利。

## 7.5 三个阶段的"不学清单"总表

| 阶段 | 该会 | 明确不学 |
|------|------|---------|
| 入门 | AI 编程工作流、存量技能变现、基础收款 | 新框架、新语言、"以后可能用"的一切 |
| 中阶 | 模板建站流水线、部署链路、统计三件套、支付接入 | 自建基础设施、过度工程、手搓已有模板的东西 |
| 高阶 | API 化架构、业务评测集、模型成本工程 | 追逐每个新模型、为炫技重写能跑的系统 |

技术这条线到此为止。你会发现整章的主旋律是"少学、快上、买现成的"，因为中阶以后真正的瓶颈从来不是技术，是流量。下一章就讲它。
