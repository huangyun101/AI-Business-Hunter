# 第十章 信息源与情报系统：持续找到下一个机会

第一个项目可以靠运气撞上，第二个、第三个靠什么？我在各类社群里观察那些持续拿结果的人，发现他们和"做成过一次的人"最大的区别是：**机会对他们来说是流水线的产出，不是天上掉的馅饼**。这一章讲两件事：值得长期订阅的信息源清单，以及怎么用你的编程能力把"刷信息"升级成"情报系统"。

## 10.1 先建立信息观：你找的不是"点子"，是"信号"

新手刷信息的姿势是找点子："有什么项目可以做？"老手扫信息的姿势是找信号：**谁在为什么东西付钱，付得越来越多还是越来越少**。第三章讲过的原则在这里升级成日常习惯：只追踪被钱验证过的信息。一个 Reddit 帖子的抱怨是弱信号，一个 Toolify 榜单上收入上涨的产品是强信号，一个刚完成 Stripe 收入验证的独立产品是最强信号。

## 10.2 信息源清单：按用途分类

**看"什么在赚钱"（选品/需求侧）：**
- Indie Hackers 产品库：重点看有 Stripe 收入验证的产品（哥飞四步挖掘法的起点）；
- Toolify：AI 工具的流量与收入榜单。"不停学习榜单上的赚钱产品"，是那位 AdSense 赚了十几万美金的圈友的日常动作；
- AppSumo：SaaS 折扣平台，什么产品卖得动一目了然；
- 点点数据/七麦（国内 App）、App Store 关键词：做 App 方向的对应物。

**看"什么在被搜索"（流量侧）：**
- Google Trends、Similarweb、Semrush、Ahrefs：竞对流量和关键词的循环挖掘（第八章）；
- 国内平台的搜索下拉词、聚光关键词包：有圈友把 100 个关键词丢进聚光拉出搜索指数，再让 AI 整理成完整的关键词热度报告；
- 爱搜（aidso.com）这类关键词监测工具：持续盯自己和对手的词。

**看"用户在抱怨什么"（痛点侧）：**
- Reddit、竞品的差评区、目标人群聚集的微信群/QQ 群：配合那段"把用户评价全部抓下来喂 AI"的产品分析提示词，差评区就是需求清单；
- 你自己的客户：中阶以后，最值钱的信息源是已付费用户的"还想要"。

**看"同行怎么做成的"（方法论侧）：**
- 付费社群的精华复盘帖（生财有术这类社群沉淀了大量含数据、含踩坑的一手复盘，本书引用的案例几乎全部来自这类帖子）；
- 海外的 build in public 社区、HN 的 Show HN 版块。

**信息源的纪律**：每类订阅两三个就够。信息源超过十个的人，通常在用收藏代替行动。第二章的老毛病换了个马甲。

## 10.3 从"刷"到"系统"：程序员的独门优势

普通人只能手动刷，你可以让代码替你值班。各类社群里已经有两个成型的样板。

**样板一：个人需求雷达**。第三章提过的那位圈友，让 AI Agent 持续监听自己所在的社群聊天，把零散讨论自动整理成结构化需求分析（需求名称、用户场景、原始信号、痛点、现有解决方式、为什么还有机会、MVP 怎么做、适不适合我、下一步验证）。零散信息进，可决策情报出。

**样板二：热点产品流水线**。一位圈友的自动化工作流配置：产品 Agent 每天去 Hacker News、Product Hunt、微博找热点产品，过滤出适合做 AI 网页小工具的，做微创新设计并输出产品分析报告；开发 Agent 挑评分最高的直接开发 MVP 网页。**从发现机会到做出原型，全程无人值守**。再配合定时任务（比如每天上午 9 点自动执行），你睡觉时情报系统在上班。

搭建自己版本的最小配置，一个周末能完成：

1. **采集层**：RSS/API 拉取你选定的三五个信息源；国内平台可用影刀、八爪鱼这类可视化采集工具（有现成模板，开箱即用）；
2. **过滤层**：让 AI 按你的标准初筛，"适合一人开发、面向付费意愿人群、我的技能栈能覆盖"，标准写成 prompt 固定下来；
3. **输出层**：每天一份简报推到你的飞书/微信，格式用需求雷达那八个维度；
4. **闭环层**：每周从简报里挑一条，用第三章的 30 分钟验证流程处理掉。**没有闭环层的情报系统只是高级收藏夹。**

## 10.4 社群：最贵也最值的信息源

工具解决"知道"，社群解决"信"和"快"。第八章讲过社群作为获客渠道的用法，这里讲它作为信息源的用法。

**抄作业的正确姿势**。社群里的复盘帖自带完整的"输入-动作-数据-坑"链条。抄作业不是抄项目名，是抄决策结构：他为什么选这个切入点？他在哪一步差点放弃？他的数据合格线定在哪？第四章说过，那位新人 Summer 第一次跟着社群航海项目就做出 1.5 万，靠的不是项目多新，是作业足够详细。

**信息差的另一半是时间差**。同一个信息，社群里的一手复盘比公众号的二手解读早三个月，比短视频的三手贩卖早半年。第四章"新生态服务商"的打法（比小白早到场两周）依赖的就是这个时间差。你付的社群年费，买的是提前量。

**从潜水到冒泡是个投资决策**。再讲一遍那个样本：一位小老板在社群潜水一年只看帖，第二年发了第一篇复盘成为精华帖，"从此改变了我的人生"。转行、人脉、客户全从这篇帖子长出来。信息源用到高阶，你自己就该变成别人的信息源，那时候信息会主动来找你（这正是下一章个人 IP 的入口）。

## 10.5 分段位小结

**入门**：订阅三五个信息源，每天 20 分钟手动扫，重点练"信号/噪音"的判断力；加入一个有真实复盘的付费社群，认真抄一次作业。

**中阶**：搭最小情报流水线（采集 → AI 过滤 → 每日简报 → 每周验证一条），把找机会的时间成本从每天一小时压到每周一小时。

**高阶**：情报系统对外输出。每周把你的观察写成内容发出去，输出倒逼输入质量，还顺手完成了获客和 IP 建设。你的情报系统本身也可以是产品：卖简报、卖数据、卖监测服务，又一个卖水人的位置。

信息源解决"下一个机会在哪"，下一章解决"怎么让同样的时间产出十倍的结果"：杠杆。
