【从"我会写代码"到"我能靠代码赚到工资以外的钱"，中间缺的不是技术，是一张地图】

程序员做副业最大的坑，不是代码写不出来，是所有代码都写完了才发现没人要。
我在生财泡了大半年，翻了几百份精华帖，发现一个残酷规律：倒在做不出来的程序员很少，倒在根本不知道做什么、做出来卖给谁、收多少钱的程序员，一片一片。
这本手册就是我自己踩坑过程中的产物。我把生财里那些做出来的程序员圈友的复盘——赫兹、小耳朵、王马扎、浩然、松月——全部拆解、归类、串联成了一条路，从月入几百到月入几万，每一步都有真实案例的路径量级做参考。

手册长什么样？给几个切面感受一下：
- 不是罗列项目清单，是把程序员能做的生意归成六条主干道（接单、教学、虚拟产品、工具站、SaaS、自动化服务），每条路标清楚：卖什么、天花板在哪、第一块钱多快到手
- 第二章专门讲心理建设——"不敢开始"怎么破、"完美主义是消耗品"、"预期校准到以月为单位"。说实话这章我写的时候自己最痛，因为每个坑我都躺过
- 第三章把需求发现拆成了四个来源+一套验证框架，最有价值的是圈友的实战做法：有人让 AI Agent 当"需求雷达"监听社群聊天、有人用 Indie Hackers 收入验证数据倒推关键词
- 技术层面专门有一章，但方向反着写：每个阶段不该学什么。程序员最容易用学习代替经营，这一章就是给你省钱省时间的
- 最后两章是杠杆+避坑：自动化先上、IP 跟上、人是最后上的；方向的坑白干一年、合规的坑一夜清零、执行的坑反复内耗

这本手册适合谁：
- 会写代码，但不知道除了接外包还能做什么
- 做过一两个项目，卡在"有点收入但上不去"
- 下班后想搞点自己的东西，但每次打开电脑都不知道从哪下手
- 在生财潜水很久，看了很多案例但不知道哪个适合自己

不适合谁：想看"三天日入过万"秘籍的。手册里最真实的案例是 4 个半月、15 个站才赚到 0.01 刀。
